Key Features

  • Real-time Multi-Guard Tracking: Independent tracking for each guard with confidence scoring
  • Action Classification: Recognizes Hands Up, Hands Crossed, T-Pose, Sitting, Standing, and more
  • Multi-Language Support: Hindi, English, Marathi, and Gujarati
  • Fugitive Detection: Instant detection of pre-specified persons with immediate audio alerts
  • Dark Mode Support: CLAHE enhancement for night/low-light tracking
  • CSV Event Logging: Complete event history with timestamps and confidence scores
  • Snapshot Management: Automatic alert snapshots with configurable retention

üì¶ Installation Guide

Step 1: Install Python

  1. Download Python 3.10 from python.org
  2. Run the installer and check "Add Python to PATH"
  3. Choose "Install for all users"
  4. Verify installation:
    python --version

Step 2: Create Virtual Environment

  1. Open Command Prompt or PowerShell
  2. Navigate to project directory:
    cd D:\CUDA_Experiments\Git_HUB\Nirikhsan_Web_Cam\V2
  3. Create virtual environment:
    python -m venv venv
  4. Activate virtual environment:
    # Windows
    venv\Scripts\activate
  5. Verify activation (you should see "(venv)" in the prompt)

Step 3: Install Dependencies

  1. Update pip:
    pip install --upgrade pip setuptools wheel
  2. Install core dependencies:
    pip install opencv-python==4.8.1.78
    pip install mediapipe==0.10.3
    pip install face-recognition==1.3.5
  3. Install deep learning frameworks:
    pip install tensorflow==2.13.0
    pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu117
  4. Install GUI and utilities:
    pip install customtkinter==5.2.1
    pip install pillow==10.0.1
    pip install pygame==2.5.0
  5. Install system utilities:
    pip install psutil==5.9.5
    pip install pydub==0.25.1

Step 4: Download Project Files

  1. Clone or download the repository
  2. Extract to your desired location
  3. Ensure these directories exist or will be created:
    • guard_profiles/ - Guard face images
    • pose_references/ - Guard pose JSONs
    • capture_snapshots/ - Captured photos
    • alert_snapshots/ - Alert moment photos
    • logs/ - Session and event logs
    • audio_files/ - Alert audio files

Step 5: Run the Application

  1. Navigate to project directory in terminal/PowerShell
  2. Activate virtual environment if not already done
  3. Run the main script:
    python Basic+Mediapose_v2.py
  4. The GUI window should open with "Camera Feed Offline"
‚úì Installation Complete! You're ready to use Niraakshan. Proceed to the Setup section for configuration.

üîß Troubleshooting

Camera Issues

Problem: Camera feed shows "Camera Feed Offline"

  • ‚úì Check camera is connected and powered on
  • ‚úì Test camera with Windows Camera app
  • ‚úì Try different camera index if multiple cameras available
  • ‚úì Restart application and re-select camera
  • ‚úì Update camera drivers

Problem: Camera feed is laggy/freezing

  • ‚úì Close other applications using camera (Zoom, Teams, etc.)
  • ‚úì Check camera USB cable connection (if USB camera)
  • ‚úì Reduce frame skip interval in config.json (currently 2, try 3-4)
  • ‚úì Disable night mode if enabled
  • ‚úì Check system RAM availability (should have 4GB+ free)

Face Recognition Issues

Problem: "Image must contain exactly one face"

  • ‚úì Ensure only one person is visible in the image
  • ‚úì Move background clutter out of frame
  • ‚úì Use clear, well-lit image
  • ‚úì Guard should face camera directly
  • ‚úì Minimum face size: 50x50 pixels

Problem: Guards not detected during tracking

  • ‚úì Verify guard profile photos contain good quality face
  • ‚úì Check camera lighting (minimum 300 lux recommended)
  • ‚úì Reduce face_recognition_tolerance in config.json (e.g., 0.4 ‚Üí 0.35)
  • ‚úì Take new guard profile photos in same lighting as tracking location
  • ‚úì Try uploading photo from tracking location itself

Performance Issues

Problem: High CPU/Memory usage

  • ‚úì Check FPS - should be 20-30 FPS
  • ‚úì Reduce number of tracked guards (max 3-5 per system)
  • ‚úì Enable frame skipping: "enable_frame_skipping": true
  • ‚úì Increase frame_skip_interval: 2 ‚Üí 3-4
  • ‚úì Reduce camera resolution in config if supported
  • ‚úì Close browser/media players to free memory
  • ‚úì Install NVIDIA CUDA for GPU acceleration (10x faster)

Problem: Audio alerts not playing

  • ‚úì Check system volume and speaker connections
  • ‚úì Verify audio file exists: audio_files/siren.mp3
  • ‚úì Test audio: Run pygame.mixer test
  • ‚úì Install ffmpeg for MP3 support: pip install pydub
  • ‚úì Windows may default to winsound (beeps) if pygame fails

Pose Detection Issues

Problem: False action alerts (guards not performing action)

  • ‚úì Increase alert timeout to avoid false positives
  • ‚úì Verify pose detection: check logs for confidence scores
  • ‚úì Ensure full body is visible in camera frame
  • ‚úì Check lighting - poor lighting reduces pose accuracy
  • ‚úì Multiple guards: resolve overlapping bounding boxes

Problem: Pose not detected (guards not recognized)

  • ‚úì Ensure full body is visible (head to feet)
  • ‚úì Reduce min_detection_confidence in config.json (0.5 ‚Üí 0.3-0.4)
  • ‚úì Improve lighting (500+ lux recommended)
  • ‚úì Use night mode for low-light environments
  • ‚úì Try different camera angle/position

Software Errors

Problem: "ModuleNotFoundError: No module named 'mediapipe'"

pip install mediapipe --upgrade

Problem: "ImportError: DLL load failed"

  • ‚úì Update CUDA and cuDNN if using GPU
  • ‚úì Reinstall tensorflow: pip install tensorflow --upgrade
  • ‚úì Update Windows: Windows Update > Check for updates

Problem: "Unicode encoding error" on Windows console

  • ‚úì App automatically converts Unicode to ASCII for console
  • ‚úì Check logs/ directory for full Unicode logs
  • ‚úì Not a fatal error - app continues running

Configuration Issues

Problem: "Config load error. Using defaults."

  • ‚úì Verify config.json exists in project root
  • ‚úì Check JSON syntax (use JSONLint.com)
  • ‚úì Ensure correct directory paths in config
  • ‚úì Verify file permissions (read/write)
